# BerryNext 의사결정 보조 구현 방향

## 한 문장

BerryNext는 현재 센서 상태, 단기 환경 예측, 시나리오 비교, 문헌/매뉴얼 기반 재배 임계값을 결합해 오늘의 딸기 온실 관리 작업을 추천하는 스마트팜 의사결정 보조 시스템이다.

## 핵심 포지션

BerryNext는 농업인의 작업 선택을 작업이력 라벨로 지도학습하는 모델이 아니다.
BerryNext는 온실 구동기를 직접 움직이는 자율 제어 시스템도 아니다.
BerryNext의 출력은 사람이 검토하는 작업 추천, 근거, 주의사항, 보수적 fallback 정보다.

현재 임계값은 문헌/매뉴얼 기반의 초기 가정으로 다룬다. 로컬 농장, 품종, 배지, 계절, 센서 보정값으로 검증되기 전까지는 검증된 재배 처방이나 확정적 예측이라고 표현하지 않는다.

## 최종 아키텍처

```text
Sensor Data
→ Current State Builder
→ Environmental Predictor
→ Scenario Simulator
→ Threshold / Literature Rule Engine
→ Work-Need Scorer
→ Recommendation Generator
```

| 단계 | 역할 |
| --- | --- |
| Sensor Data | 온도, 습도, VPD, 일사, CO2, 지습, EC/pH, 배액률, 외기/강우, 제어 이력 등을 입력으로 받는다. |
| Current State Builder | 원천 API, 엑셀, JSON을 현재 온실 상태 계약으로 정규화한다. |
| Environmental Predictor | 다음 몇 시간의 온도, 습도, VPD, 일사, 지습, EC 등 환경 변화량을 예측한다. |
| Scenario Simulator | 관수, 환기, 차광, 보온, EC 점검 같은 후보 action의 방향성 효과와 위험을 비교한다. |
| Threshold / Literature Rule Engine | 문헌/매뉴얼 기반 임계값과 주의사항을 구조화해 추천 근거로 제공한다. |
| Work-Need Scorer | 현재 상태, 예측 상태, 시나리오, 규칙 hit를 모아 작업 필요도 점수를 계산한다. |
| Recommendation Generator | 사람이 검토할 오늘의 추천 작업, 이유, 근거, 안전 플래그, fallback 여부를 만든다. |

## Level 1 핵심 작업

| 작업 | 판단 신호 | 출력 |
| --- | --- | --- |
| 관수 | 지습, VPD, 일사, 예측 수분 변화, 최근 관수 간격 | 관수 검토 필요도, 수분 부족/과습 주의, 확인할 센서 |
| EC/양액 조정 검토 | 급액 EC, 배액 EC, 근권 EC, pH, 배액률 | EC/양액 점검 필요도, 염류 집적/희석 주의, 조정 전 확인 항목 |
| 환기 | 습도, VPD, 외기온, 강우, 엽면젖음 proxy, 환기 이력 | 환기/제습 검토 필요도, 냉해/건조 위험 |
| 차광 | 내부 온도, 일사, VPD, 고온 지속 예측 | 차광/열부하 완화 검토 필요도, 과도 차광 주의 |
| 보온/난방 검토 | 내부 온도, 외기온, 야간 저온 예측, 생육단계 | 보온/난방 점검 필요도, 에너지/습도 상승 주의 |

## Level 2 보조 알림

다음 항목은 핵심 작업 추천이 아니라 보조 alert다.

| 보조 알림 | 범위 |
| --- | --- |
| 병해 위험 예찰 알림 | 고습, 낮은 VPD, 강우, 엽면젖음, 병반 proxy가 있을 때 현장 예찰 우선순위를 알린다. 실제 병해 발생 예측이나 방제 처방으로 표현하지 않는다. |
| 수확 가능성 알림 | 착색, 과실 수, 고온/강우 품질 위험을 근거로 수확 가능성을 모니터링한다. 검증된 수확 시기 예측으로 표현하지 않는다. |
| 적엽 검토 알림 | 수관 밀도, 습도, 최근 적엽 간격을 근거로 보수적 검토를 알린다. 검증된 적엽 추천이나 생육 처방으로 표현하지 않는다. |

## 작업이력 사용 원칙

작업이력은 지도학습 label이 아니다.

사용 가능:

- 최근 작업 이후 경과일을 이용한 안전 간격
- 과도한 반복 작업을 막는 cooldown
- "최근 관수 후 센서 반응을 확인해야 함" 같은 설명
- 예찰, 수확, 적엽 보조 알림의 보수적 감점 또는 주의사항

사용 금지:

- 농업인의 과거 작업 선택을 정답 label로 삼아 직접 분류 모델을 학습했다고 주장
- 시나리오 시뮬레이션 결과를 가짜 작업 label로 만들어 supervised training에 사용
- 작업이력만으로 병해, 수확, 적엽을 검증된 예측이라고 주장

## Environmental Predictor 로드맵

GAM은 직접 작업을 분류하는 모델이 아니라 단기 환경 변화량 예측 후보 중 하나다.

| 단계 | 예측기 | 설명 |
| --- | --- | --- |
| v0 | no-change baseline | 현재 상태가 단기적으로 유지된다고 보는 기준선 |
| v1 | rolling delta baseline | 최근 변화량 평균으로 다음 변화량을 추정 |
| v2 | linear regression | 환경 피처와 시간 피처로 단기 변화량을 선형 예측 |
| v3 | GAM | 비선형이지만 설명 가능한 환경 변화량 예측 |
| v4 | LSTM 또는 Transformer | 충분한 데이터와 검증이 있을 때만 장기/복합 시계열 후보로 검토 |

모든 예측기는 `PredictionResult` 계약과 confidence gate를 통과해야 추천 점수에 영향을 줄 수 있다. 약한 모델, 부족한 데이터, baseline보다 나쁜 검증 결과는 문헌/매뉴얼 규칙으로 fallback한다.

## 시나리오 시뮬레이션 원칙

Scenario Simulator는 후보 action을 비교하는 설명 도구다.

- 출력은 방향성 효과, 잠재 이점, 잠재 위험, 가정, 근거를 제공한다.
- 검증된 causal simulator라고 표현하지 않는다.
- 온실 구동기 제어 명령을 발행하지 않는다.
- 시뮬레이션 결과를 supervised training용 fake label로 사용하지 않는다.

## 데이터 계층

| 데이터 그룹 | 주요 변수 | 활용 |
| --- | --- | --- |
| 내부 환경 | 온도, 습도, VPD, CO2, 일사량 | 현재 상태, 단기 환경 예측, 환기/차광/보온 판단 |
| 외부 기상 | 외기온, 강우, 일사, 습도 | 환기, 보온, 병해 예찰 proxy, 시나리오 가정 |
| 양액/근권 | EC, pH, 급액량, 배액량, 배지 수분 | 관수와 EC/양액 조정 검토 |
| 구동기 이력 | 천창, 측창, 차광, 팬, 난방, 양액기 | 현재 상태 설명, 안전 간격, 시나리오 가정 |
| 생육/이미지 proxy | 과실 수, 착색 proxy, 엽밀도, 병반 proxy | Level 2 보조 알림 |
| 작업이력 | 최근 관수, 양액 조정, 환기, 차광, 난방, 예찰, 수확, 적엽 | safety spacing, cooldown, 설명 |

## 구현 원칙

- 추천 결과에는 항상 이유, 근거, 위험, 안전 플래그를 포함한다.
- actuator 예제는 API 사용 예시로 유지하되 추천 출력과 분리해서 설명한다.
- `disease_environment_risk_proxy`, `harvest_monitoring`, `leaf_removal_caution`은 Level 2 auxiliary alert로만 노출한다.
- Threshold/Literature Rule Engine의 수치는 로컬 검증 전까지 provisional rule로 표시한다.
- 모델이 추천을 주도했다는 표현은 confidence gate를 통과한 예측에만 제한한다.

## 발표용 한국어 요약

BerryNext는 딸기 스마트팜의 센서 데이터를 현재 상태로 정리하고, 단기 환경 변화를 예측한 뒤, 관수, 양액 EC, 환기, 차광, 보온 같은 관리 후보를 문헌 기반 기준과 시나리오 비교로 평가하는 설명 가능한 의사결정 보조 시스템입니다. 이 시스템은 농가의 과거 작업을 정답 라벨로 학습하거나 온실을 자율 제어하지 않으며, 병해·수확·적엽은 검증된 예측이 아니라 예찰과 점검을 돕는 보조 알림으로만 제공합니다.
