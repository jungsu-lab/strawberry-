# BerryNext AI 구현 방향

## 한 문장

BerryNext AI는 생육단계, 온실환경, 이미지/예찰, 작업이력을 통합해 딸기 농가의 "오늘 할 일"을 추천하는 농작업 의사결정 AI이다.

## 핵심 포지션

기존 스마트팜 AI는 온도, 습도, CO2 같은 환경 제어에 집중되어 있다.
BerryNext AI는 여기서 한 단계 더 나아가 농업인이 매일 판단하던 방제, 관수, 수확, 적엽 시점을 데이터 기반으로 보조한다.

즉, 목표는 온실을 자동으로 움직이는 AI가 아니라 농작업의 우선순위와 실행 시점을 계산하는 AI이다.

## MVP 범위

기능을 넓게 벌리기보다 다음 3개 엔진을 우선 구현한다.

| 우선순위 | 엔진 | 출력 |
| --- | --- | --- |
| 1 | 오늘 할 일 추천 | 방제, 관수, 수확, 적엽의 우선순위 |
| 2 | 관수/양액 의사결정 | 다음 관수 시점, 관수 필요도, EC/pH 조정 주의 |
| 3 | 병해 위험도 예측 | 위험도, 예찰 필요 여부, 방제 준비 권장 |
| 4 | 수확 시기/생산량 예측 | 수확 추천 여부, 예상 수확 가능성 |
| 5 | 적엽/수관 관리 | 잎 밀도, 고습, 작업 간격 기반 적엽 검토 |

정밀 이미지 분석과 모델 기반 생산량 예측은 확장 기능으로 둔다.

## 시스템 구조

1. `SmartFarmKoreaClient`
   - 스마트팜코리아 데이터마트 API에서 시설, 환경, 제어, 생육 데이터를 가져온다.
2. 데이터 정규화 계층
   - API 응답, 대회 센서 데이터, 이미지 분석 결과를 `GreenhouseSnapshot` 형태로 맞춘다.
3. `BerryNextDecisionEngine`
   - 관수/양액, 병해, 수확 추천을 점수화한다.
4. `DailyFarmWorkDecisionEngine`
   - 생육단계, 온실환경, 이미지/예찰, 작업이력을 `FarmWorkContext`로 합쳐 오늘 할 농작업을 정렬한다.
5. 안전 제약
   - 과습, 고습, 급격한 제어 변경, 모델 신뢰도 부족 상황에서 보수적으로 추천한다.
6. 대시보드 또는 리포트
   - 오늘 할 작업, 추천 이유, 예상 효과, 주의사항을 농업인이 이해하기 쉽게 보여준다.

## 구현 원칙

- 처음에는 규칙 기반 점수화로 투명하게 만든다.
- 실제 데이터가 확보되면 각 점수 함수를 LightGBM, XGBoost, 시계열 모델, 이미지 모델로 교체한다.
- 추천 결과에는 항상 이유와 안전장치를 포함한다.
- 모델 출력은 직접 제어 명령이 아니라 작업 추천으로 시작한다.
- 데이터는 딸기 전용 입력만 사용한다. 혼합 작물 데이터는 딸기 행만 추출하거나 제외한다.
- 추천 결과는 `관수`, `방제/예찰`, `수확`, `적엽`을 같은 `DailyFarmWorkPlan` 안에서 비교 가능하게 만든다.

## 발표용 연결 문장

현재 첨단 디지털온실은 이미 다양한 센서와 구동기를 갖춘 데이터 기반 재배 환경이다.
그러나 이 데이터를 실제 농작업 판단으로 연결하는 의사결정 체계가 핵심 과제로 남아 있다.
BerryNext AI는 이 간극을 해결하기 위해 환경, 양액, 이미지, 에너지 데이터를 통합하고 딸기 재배의 다음 행동을 추천하는 AI 의사결정 엔진을 설계한다.
