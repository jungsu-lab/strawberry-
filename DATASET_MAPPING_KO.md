# 딸기 전기/펠릿 데이터셋 매핑

이 프로젝트는 스마트팜코리아에서 내려받은 `딸기 전기`, `딸기 펠릿` 폴더의 엑셀 데이터를 BerryNext 농작업 의사결정 AI의 입력으로 사용한다.

## 폴더 구성

두 폴더는 같은 이름의 8개 `.xlsx` 파일을 가진다.

| 파일 | 용도 |
| --- | --- |
| `시설원예_생육정보_2026-06-23.xlsx` | 주차별 생육 상태 |
| `시설원예_시간기준_환경정보_2026-06-23.xlsx` | 시간 단위 온실환경 |
| `시설원예_일기준_환경정보_2026-06-23.xlsx` | 일 단위 온실환경 |
| `시설원예_주기준_환경정보_2026-06-23.xlsx` | 주 단위 온실환경 |
| `시설원예_월기준_환경정보_2026-06-23.xlsx` | 월 단위 온실환경 |
| `시설원예_시간대별_환경정보_2026-06-23.xlsx` | 야간/주간 등 시간대 평균 환경 |
| `시설원예_시간기준_제어정보_2026-06-23.xlsx` | 시간 단위 제어 이력 |
| `딸기_도매시장가격정보_일별.xlsx` | 일별 도매시장 가격 |

## 공통 입력 매핑

| BerryNext 입력 | 데이터셋 컬럼 |
| --- | --- |
| 내부 온도 | `내부-내부온도` |
| 내부 습도 | `내부-내부습도` |
| CO2 | `내부-내부CO2` |
| 일사량 | `외부-외부일사량`, 보조로 `내부-내부일사량` |
| 강우 신호 | `외부-강우감지` |
| 환기/개도 | `제어-좌측일중천창 개도율`, `제어-우측일중천창 개도율`, `제어-좌측일중측창 개도율`, `제어-우측일중측창 개도율` |
| 생육단계 추정 | `주차`, `화방착과수1(개)`, `화방착과수2(개)`, `화방착과수3(개)` |
| 적엽/수관 신호 | `엽수(개)`, `엽장(mm)`, `엽폭(mm)` |
| 수확 가능성 | `화방착과수1(개)`, `화방착과수2(개)`, `화방착과수3(개)`, 도매가격 추세 |

## 전기/펠릿 차이

전기와 펠릿은 공통 환경 컬럼이 많지만, 센서 구성이 완전히 같지 않다.

| 구분 | 주요 차이 |
| --- | --- |
| 딸기 전기 | `토양-지습`, `양액-지습`, `내부-내부일사량`, `이슬점온도`, `절대습도`가 있다. |
| 딸기 펠릿 | `토양-토양EC`, `토양-토양PH`, `양액-(양액)배액EC`, `양액-(양액)배액PH`가 있다. |
| 제어정보 | 전기는 보일러, 유동팬, CO2 발생장치 등 제어 컬럼이 많고 결측도 많다. 펠릿은 창 개도율과 차광커튼 중심으로 더 단순하다. |

따라서 모델 입력은 다음처럼 나눈다.

1. 공통 특징: 온도, 습도, CO2, 외부일사량, 외부온도, 강우감지, 풍속, 천창/측창 개도율
2. 전기 전용 특징: 지습, 내부일사량, 이슬점, 절대습도, 보일러/팬/CO2 제어 여부
3. 펠릿 전용 특징: 토양 EC, 토양 pH, 배액 EC, 배액 pH
4. 비교 특징: 난방방식 라벨 `electric` 또는 `pellet`

## 오늘 할 일 의사결정 구조

`DailyFarmWorkDecisionEngine`에는 하루 단위로 정규화한 입력을 넣는다.

| 농작업 | 판단 근거 |
| --- | --- |
| 관수/양액 | 고온, 고일사, 낮은 지습, EC/pH 이상, 최근 관수 경과일 |
| 방제/예찰 | 고습, 낮은 VPD, 강우감지, 환기 부족, 최근 예찰/방제 경과일 |
| 수확 | 생육 주차, 화방착과수, 가격 추세, 최근 수확 경과일 |
| 적엽 | 엽수, 엽장/엽폭, 고습, 환기 부족, 최근 적엽 경과일 |

현재 폴더에는 직접 이미지 파일이 없으므로 `이미지/예찰` 입력은 우선 생육정보의 엽수/엽장/엽폭/화방착과수와 환경 기반 병해 위험 신호로 대체한다. 나중에 이미지 파일이나 병반 검출 결과가 추가되면 `ImageGrowthSignals`의 `leaf_density`, `disease_spot_ratio`, `ripe_fruit_ratio`에 연결한다.

## 전처리 단위

초기 MVP는 일 단위 데이터셋을 기준으로 만든다.

1. `일기준_환경정보`에서 날짜별 온실환경을 가져온다.
2. `생육정보`는 가장 가까운 조사일을 해당 날짜에 forward-fill한다.
3. `시간기준_제어정보`는 날짜별 평균/최대 개도율과 장비 작동 여부로 집계한다.
4. 작업이력은 초기에는 규칙 기반으로 계산한다.
5. 결과는 `FarmWorkContext`로 변환해 `DailyFarmWorkDecisionEngine.plan_today()`에 넣는다.

## 모델 확장 방향

처음에는 규칙 기반으로 오늘 할 일을 설명 가능하게 추천한다. 데이터가 더 쌓이면 작업별 점수 함수를 별도 모델로 바꾼다.

| 단계 | 방법 |
| --- | --- |
| MVP | 규칙 기반 점수화 |
| 2단계 | 전기/펠릿 비교 리포트, 날짜별 추천 로그 생성 |
| 3단계 | LightGBM/XGBoost로 관수, 방제, 수확, 적엽 필요도 예측 |
| 4단계 | 이미지/예찰 결과를 결합한 멀티모달 의사결정 |
